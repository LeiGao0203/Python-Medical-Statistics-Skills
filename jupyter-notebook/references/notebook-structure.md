# Notebook Structure

Notebook scaffolds should make intent, execution, and interpretation easy to review.

## Core Sections

1. Title
2. Objective or learning goals
3. Setup and imports
4. Data source or worked example
5. Analysis or lesson body
6. Interpretation
7. Limitations or common pitfalls
8. References and session notes

## Cell Guidance

- Keep markdown cells short and purposeful.
- Use code cells for one logical step at a time.
- Place imports near the top and avoid hidden setup later in the notebook.
- Use stable relative paths from the project root.
- Prefer tables and plots that are labeled enough to stand alone.

## Output Discipline

- Clear exploratory noise before sharing a notebook.
- Leave important result outputs visible when they support the narrative.
- Avoid storing protected health information in notebooks or outputs.
- Use synthetic or de-identified examples for tutorials.
